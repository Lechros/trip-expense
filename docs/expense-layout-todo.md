# 지출 탭 — UI 레이아웃만 (기능 없음)

기능·로직 연동 없이 **화면 구조·배치**만 구현할 부분.

---

## 1. 필터 UI 레이아웃

- **위치**: "필터" 버튼 클릭 시 열리는 화면(바텀시트 권장).
- **구성**:
  - 헤더: "필터" 제목.
  - 본문:
    - **결제자**: 라벨 + 선택 영역(플레이스홀더 또는 멀티 선택 목록 형태만).
    - **수혜자**: 라벨 + 선택 영역(동일).
    - **날짜**: 라벨 + 기간 입력 영역(시작일·종료일 플레이스홀더만).
    - **통화**: 라벨 + 통화 선택 영역(단일 선택 플레이스홀더만).
  - 푸터: [초기화] [적용] 버튼.
- **동작**: 열기/닫기만. 선택 값 저장·목록 필터링·API 연동 없음.

---

## 2. 삭제된 항목 보기 시 목록 레이아웃

- **현재**: "삭제된 항목 보기" 토글만 있고, 목록은 항상 동일.
- **추가할 레이아웃**:
  - 토글 **켰을 때**: 목록 위에 **"삭제된 항목"** 섹션 헤더(또는 구분선) 표시, 그 아래 리스트 영역.  
    삭제된 항목이 없을 때를 위한 **빈 상태** 영역: "삭제된 항목이 없습니다" 문구 + (선택) 일러스트/아이콘.
  - 토글 **껐을 때**: 기존처럼 "삭제된 항목" 섹션 미표시.
- **동작**: 실제 삭제/복구 로직 없음. `showDeleted === true`일 때 위 블록이 보이기만 하면 됨.

---

## 3. 제외(레이아웃만 해당 없음)

- **가상 리스트**: 스크롤/성능 구현에 가까우므로 레이아웃 정리 범위에서 제외.
- **Soft delete 데이터 구조**: 기능 구현이므로 제외.
